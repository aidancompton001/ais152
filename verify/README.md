# verify/ — Верификация цифрой (Закон 21)

> **«ЦИФРЫ не обманешь.»** Заявление «задача выполнена» подтверждается машиной, не агентом.

## Зачем

Агент — единственный свидетель своей работы и сам же пишет отчёт. Текст «тесты прошли / готово» может быть галлюцинацией. `verify.py` убирает агента из цепочки доверия: скрипт читает **реальное** состояние проекта и печатает вердикт, который агент подделать не может.

## Как пользоваться

1. Скопируй `acceptance.example.json` → `verify/acceptance.json` в корне проекта
2. Впиши КАЖДЫЙ критерий приёмки задачи как исполняемую проверку (`cmd` + `expect_*`)
3. Запусти:
   ```
   python verify/verify.py
   ```
4. Скрипт прогоняет все проверки, печатает `passed/total`, процент, exit code
5. Вставь **ДОСЛОВНЫЙ** вывод в отчёт CEO

## Типы проверок

| Поле | Значение |
|------|----------|
| `expect_exit` | команда вернула этот exit code |
| `expect_equal` | stdout (trimmed) РОВНО равен значению |
| `expect_min` | stdout как число `>=` значения |
| `expect_contain` | stdout содержит подстроку |

## Правила (Закон 21)

- Задача завершена ТОЛЬКО при `RESULT: N/N PASSED (100%)`, exit 0
- В отчёт — ДОСЛОВНЫЙ вывод. Пересказ запрещён = страйк
- Каждый `cmd` — реальная команда, читающая реальное состояние (`test -f`, `npm test`, `curl`/`fetch`, `grep -c`, `node -e`, `sha256sum`)
- Полная сводка раннера — не отфильтрованное подмножество
- Задачи без машинного результата (тексты, методология) — структурный пруф (`git diff --stat`, hash, `wc -l`) с честной пометкой «структурный пруф»
