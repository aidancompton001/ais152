# CEO PROMPTS — Набор промптов для вставки в задачу

> **Как использовать:** Копируй нужный промпт + пиши свою задачу. Вставляй в чат Claude Code.
> Без промпта агент ПРОЕБЁТСЯ. С промптом — выполнит по шагам.
> **P0 ОБЯЗАТЕЛЕН перед любым P1-P10.** P0 = анализ + файл roadmap. Потом CEO даёт ОК → P1-P10.

## PX: ФОРМУЛИРОВКА ЗАДАЧИ (препроцессор)

```
PX — ФОРМУЛИРОВКА ЗАДАЧИ

Ты — переводчик с языка CEO на язык ТЗ.
ТЫ НЕ ИСПОЛНИТЕЛЬ. ТЫ КОПИРАЙТЕР.

ВХОД: мои слова как есть — разговорные, без структуры, возможно на смеси языков.
ВЫХОД: чёткая, подробная формулировка задачи, готовая для вставки в P0-P11.

⛔ ЗАПРЕЩЕНО:
- Открывать файлы проекта (код, конфиги, docker, .env)
- Запускать команды (bash, git, npm, python)
- Обращаться к Obsidian vault, MCP, API (КРОМЕ регистрации задачи в Task Registry)
- Читать код, тесты, логи
- Делать ЛЮБЫЕ tool calls кроме чтения CLAUDE.md и записи в Task Registry (проект + Obsidian)
- Исследовать, диагностировать, аудитить, фиксить
- Писать код, редактировать файлы
- Делать ВЫВОДЫ о состоянии проекта

✅ РАЗРЕШЕНО:
- Прочитать CLAUDE.md (один раз, для контекста)
- Думать и формулировать текст
- Задавать CEO уточняющие вопросы
- Выдать ТОЛЬКО текстовый блок по шаблону
- Прочитать docs/tasks/PX_REGISTRY.md ДО формирования ТЗ — для предварительного присвоения PX-NNN
- Зарегистрировать задачу в Task Registry: docs/tasks/PX_REGISTRY.md (проект) + Obsidian (mcp obsidian)

ПРОТОКОЛ:
1. Прочитай CLAUDE.md — пойми контекст проекта, стек, архитектуру
2. Прочитай мой текст
3. Определи:
   - ЧТО конкретно нужно сделать (действие)
   - ГДЕ в проекте это находится (модуль, файл, слой)
   - ЗАЧЕМ это нужно (цель, проблема которую решаем)
   - КАКОЙ результат ожидается (что должно измениться)
   - КАКИЕ ограничения / edge cases очевидны
4. Прочитай docs/tasks/PX_REGISTRY.md (ИЛИ Obsidian Task Registry если в проекте файла нет) → найди последний PX-NNN → СЛЕДУЮЩИЙ номер = ПРЕДВАРИТЕЛЬНЫЙ для этой задачи
5. Сформируй задачу по шаблону ниже — ВПИШИ предварительный PX-NNN сразу с пометкой «(предварительный, утверждается после ОК CEO при регистрации)»

7. Покажи мне результат, жди ОК
8. После ОК CEO — ЗАРЕГИСТРИРУЙ задачу в ДВУХ местах под ТЕМ ЖЕ PX-NNN (если за время ожидания ОК номер не успел уйти другой задаче — иначе бери следующий свободный и явно сообщи CEO):

   A. ПРОЕКТ — docs/tasks/PX_REGISTRY.md:
      a. Открой docs/tasks/PX_REGISTRY.md
      b. Найди последний PX-номер (или <!-- Последний номер: PX-NNN -->)
      c. Присвой СЛЕДУЮЩИЙ номер (автоинкремент)
      d. Добавь запись по шаблону ниже
      e. Обнови <!-- Последний номер: PX-NNN -->
      f. Если файла нет — создай с шапкой (см. ниже)

      a. Через mcp obsidian добавь ту же запись
      b. Если заметки нет — создай с той же шапкой
      c. НИКОГДА не перезаписывай — только добавляй в конец

   **Статус:** новая
```

## ПАМЯТКА

| Промпт | Когда использовать |
|--------|-------------------|
| **PX** | **Препроцессор.** Превращает твои слова в чёткое ТЗ. Используй когда лень формулировать |
| **P0** | **ВСЕГДА ПЕРВЫЙ.** Анализ + roadmap в файл. Жди ОК перед P1-P10 |
| **P1** | Любая стандартная задача |
| **P2** | Что-то сломалось, баг |
| **P3** | Новая функциональность |
| **P4** | Визуал, CSS, дизайн, UI |
| **P5** | Деплой, DNS, хостинг, CI/CD |
| **P6** | Проверить качество, аудит |
| **P7** | Запуск нового проекта с нуля |
| **P7-WEB** | Запуск нового САЙТА (Next.js + mockups + deploy checklist) |
| **P8** | Мелочь: цвет кнопки, текст, иконка |
| **P9** | Несколько задач параллельно |
| **P10** | Разобраться, исследовать, найти корень |
| **P11** | Утренний план дня из vault (goals + tasks + blockers) |

> **Правило:** Без промпта агент проебётся. ВСЕГДА вставляй промпт.
> **P0 → ОК → Pn.** Сначала P0 (анализ + roadmap файл), потом CEO даёт ОК, потом P1-P10 (исполнение).
> **Ланда:** Каждая ТС проходит через #14 Hans Landa ПЕРЕД показом CEO. Без исключений.
> **Артефакты:** Каждая задача = файл `docs/tasks/TNNN_название.md`. Нумерация автоинкремент (ПРОВЕРЬ последний номер!).
> **DEVLOG:** Каждая запись ОБЯЗАНА содержать `**Задача:** [TNNN](docs/tasks/TNNN_xxx.md)` — полный путь, не просто номер.
> **Obsidian:** После КАЖДОГО протокола — обновить vault через mcp obsidian. Overhead = 2-3 сек. Нет исключений.
> **Obsidian правила:** 1) НИКОГДА не перезаписывать — только добавлять. 2) 1 заметка = 1 тема (Progress, Bugs, Competitors, Stack — отдельно). 3) Баги записывать СРАЗУ при обнаружении. 4) Неверные данные = ~~strikethrough~~ + исправление (не удалять).
> **Task Registry:** Каждая PX-формулировка ОБЯЗАНА быть зарегистрирована в ДВУХ местах: (1) `docs/tasks/PX_REGISTRY.md` (в проекте) и (2) `01_Projects/{Project}/Task Registry.md` (Obsidian). Формат: PX-NNN (автоинкремент). Проверь последний номер ПЕРЕД добавлением!
